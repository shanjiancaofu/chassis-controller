# 硬件与接线

本文集中记录 chassis-controller 当前使用的硬件器件、STM32G474 引脚分配、外设资源和实物接线。硬件变更先更新 CubeMX `.ioc`，确认无冲突后再同步本文。

## 电源与安全

- 板级调试阶段由开发板 USB-C 供电，D157B 由独立电池连接 `Vin+` 和 `GND`，输入范围按模块资料为 5.5 V 至 17 V；两者只通过公共 `GND` 和控制信号连接。
- STM32、AT8236 和 CAN 收发器必须共地。未共地时不得连接 PWM、编码器、ADC 或 CAN 逻辑信号。
- 首次电机测试必须架空车轮，限制 PWM 到 TIM8 ARR 的约 10%，先验证急停和零输出，再测试正反转。
- `PD2` 急停当前只会锁存故障并清零四路 PWM，不等同于切断电机电源的独立硬件急停。
- ADC 和普通 GPIO 输入电压不得超过 STM32 的允许范围。当前同厂配套编码器按 D157B 6P 接口直接接入；更换电机或编码器时必须重新确认 A/B 输出电平。

## AT8236 D157B 接线

### 控制与动力

| D157B | STM32/外部连接 | 说明 |
|---|---|---|
| `AIN1` | `PC6/TIM8_CH1` | 左电机输入 1 |
| `AIN2` | `PC7/TIM8_CH2` | 左电机输入 2 |
| `BIN1` | `PC8/TIM8_CH3` | 右电机输入 1 |
| `BIN2` | `PC9/TIM8_CH4` | 右电机输入 2 |
| `AO1/AO2` | 左电机动力线 | 首次低占空比测试时确认机械方向 |
| `BO1/BO2` | 右电机动力线 | 首次低占空比测试时确认机械方向 |
| `Vin+/GND` | 5.5 V 至 17 V 电机电源 | 与 STM32 共地 |
| `ADC` | `PA2/ADC1_IN3` | 输入电压的 1/11 分压采样 |

AT8236 当前采用快速衰减控制：

| 动作 | IN1 | IN2 |
|---|---|---|
| 正转 | PWM | 0 |
| 反转 | 0 | PWM |
| 滑行停止/休眠 | 0 | 0 |

左右轮以车体前进为逻辑正方向。右电机采用镜像安装，软件在 BSP 内交换 `BIN1/BIN2` 的正负方向映射；编码器 A/B 定义和方向系数保持不变。2026-07-22 实测左右电机正反转时，编码器增量符号均与逻辑方向一致。

PWM 频率为 20 kHz，比较值范围为 0 至 8499。同一电机换向时先保持至少一个 10 ms 零输出周期，再启用反向通道。任何时候不得让同一电机的两个输入同时输出有效 PWM。

### 编码器 6P 接口

| D157B | STM32 | 用途 |
|---|---|---|
| `E1A` | `PA0` | 左编码器 TIM2 CH1 |
| `E1B` | `PA1` | 左编码器 TIM2 CH2 |
| `E2A` | `PD12` | 右编码器 TIM4 CH1 |
| `E2B` | `PD13` | 右编码器 TIM4 CH2 |
| 编码器 `5V/GND` | D157B 配套接口 | 给同厂配套霍尔编码器供电 |

当前接线按 D157B 与同厂配套电机组合使用。软件通过左右编码器方向系数统一处理正负方向，不在多个业务模块中交换 A/B 定义。

### 电源电压采样

D157B 使用 10 kΩ/1 kΩ 分压，当前换算为：

```text
vin_mv = adc_raw * 3300 * 11 / 4095
```

计算使用 64 bit 中间值。2026-07-22 实测电池电压为 11.96 V，PA2 为 1.086 V，理论值为 1.087 V，分压测量误差约 0.12%；D157B 开关打开后串口电压遥测恢复正常，ADC 分压与采样链路通过实物验证。

该结果仍只用于观测，不构成过压或欠压保护。若串口持续显示 `vin_mv=0`，先确认 D157B 电源开关已经打开，再直接测量 STM32 的 PA2 引脚。

## CAN FD 接线

| MCP2562FD 模块 | 连接 |
|---|---|
| `TX` | `PB6/FDCAN2_TX` |
| `RX` | `PB5/FDCAN2_RX` |
| `VIO` | 选择 3.3 V 逻辑电平 |
| `GND` | STM32 `GND` |
| `CANH/CANL` | CAN 总线双绞线 |

收发器模块按自身要求接 5 V 电源，VIO 选择 3.3 V。120 Ω 终端电阻只在总线两个物理端点启用。首轮外部总线使用 500 kbps 仲裁速率和 2 Mbps 数据速率。

外部总线验收使用另一个支持 CAN FD 和 BRS 的节点。两节点接线如下：

```text
STM32 PB6/FDCAN2_TX -> MCP2562FD TX
STM32 PB5/FDCAN2_RX <- MCP2562FD RX
STM32 5V            -> MCP2562FD 5V
STM32 GND           -> MCP2562FD GND

MCP2562FD CANH      -> CAN FD 节点 CANH
MCP2562FD CANL      -> CAN FD 节点 CANL
MCP2562FD GND       -> CAN FD 节点 GND
```

Jetson Orin Nano 载板的 CAN 逻辑侧连接如下：

```text
Jetson CAN TX -> MCP2562FD TX
Jetson CAN RX <- MCP2562FD RX
Jetson GND    -> MCP2562FD GND
Jetson 5V     -> MCP2562FD 5V
```

Jetson 端 MCP2562FD 的 VIO 同样选择 3.3 V。CANH/CANL 使用双绞线。只有两个节点时，两端各保留一个 120 Ω 终端电阻，断电测量 CANH 与 CANL 之间应约为 60 Ω。

当前固件使用外部 normal 模式。Jetson 的仲裁段和数据段采样点均配置为 80%，否则 2 Mbps 数据阶段可能产生接收错误：

```bash
sudo modprobe can
sudo modprobe can_raw
sudo modprobe mttcan

sudo ip link set can0 down
sudo ip link set can0 type can bitrate 500000 sample-point 0.8 \
  dbitrate 2000000 dsample-point 0.8 fd on one-shot off \
  berr-reporting on restart-ms 1000
sudo ip link set can0 up
ip -details -statistics link show can0
candump can0
cansend can0 720##150494E4701000000
```

收到 STM32 的 `0x721` 响应后，再发送确认：

```bash
cansend can0 720##15041535301000000
```

此时 STM32 串口自检和 LCD 才会将外部 CAN 标记为 `PASS`。该测试不包含电机控制。

### CAN 调试命令

上面的 Jetson 配置和三步握手用于正常联调。正常基线应为 `ERROR-ACTIVE`，当前错误计数 `tx=0`、`rx=0`。`candump` 显示 `0x720` 可能只是 Jetson 控制器的本地回显；收到 STM32 返回的 `0x721`，并且 STM32 `status` 显示 `FDCAN_EXTERNAL: PASS`，才表示双向链路通过。

STM32 串口提供以下诊断命令：

```text
can status
can tx confirm
```

- `can status`：输出 FDCAN2 活动状态、仲裁段/数据段最后错误、收发错误计数、Bus-Off 和 FIFO 状态。
- `can tx confirm`：由 STM32 主动发送一帧 `0x721`，用于独立检查 STM32 TX、收发器和 Jetson RX。

典型正常输出：

```text
FDCAN_DIAG: activity=8 lec=7 dlec=0 tec=0 rec=0 passive=0 warning=0 busoff=0 restricted=0 rxfill=0 txfree=3
```

其中 `activity=8` 表示空闲，`tec/rec` 是发送/接收错误计数；`passive`、`warning`、`busoff`、`restricted` 正常时均为 `0`。`lec/dlec` 错误码含义如下：

| 值 | 含义 |
|---:|---|
| `0` | 无错误 |
| `1` | Stuff Error |
| `2` | Form Error |
| `3` | ACK Error |
| `4` | Bit 1 Error |
| `5` | Bit 0 Error |
| `6` | CRC Error |
| `7` | 自上次读取后无变化 |

### CAN 调试注意事项

- CAN 收发器的 `TX/RX` 是逻辑信号名称，不按 UART 方式交叉：`PB6/FDCAN2_TX -> 模块 TX`，`PB5/FDCAN2_RX <- 模块 RX`。接反会出现模块 TX 灯不闪、Jetson 无法收到 `0x721`、STM32 报 `Form Error`。
- 无 ACK 或接线未确认时使用单次发送，避免控制器无限重发并迅速进入 Error-Passive/Bus-Off：

```bash
sudo ip link set can0 down
sudo ip link set can0 type can bitrate 500000 sample-point 0.8 \
  dbitrate 2000000 dsample-point 0.8 fd on one-shot on \
  berr-reporting on
sudo ip link set can0 up
```

- 完成排障、进入周期控制前必须恢复 `one-shot off` 和 `restart-ms 1000`。
- `ERROR-ACTIVE` 只表示控制器当前已恢复，不代表历史报文发送成功；还要检查 `TX packets`、当前 `berr-counter` 和 STM32 的 `tec/rec`。
- 同一条失败报文可能产生大量 `bus-errors`。出现 Error-Passive 或 Bus-Off 后先停止发送并修复原因，不要反复执行 `cansend`。
- 断电检查 CANH-CANL 约为 60 Ω，CANH、CANL 和 GND 必须分别端到端连通；非隔离节点必须共地。
- 上电空闲时 CANH/CANL 均约为 2.5 V；发送时 CANH 上升、CANL 下降。STM32 侧模块 RX 和 PB5 有跳变，说明物理总线信号已经进入 MCU。
- `candump -L -e can0` 会因日志格式与错误帧显示选项冲突而报错，实时联调直接使用 `candump can0`。

## 急停接线

当前 PD2 配置为内部上拉、低电平有效、下降沿中断。直接使用常开触点：

```text
PD2 -> 急停按钮 COM
GND -> 急停按钮 NO
```

触点无极性，不连接 3.3 V 或 5 V。松开时 PD2 由内部上拉保持高电平，按下后 COM/NO 闭合并将 PD2 拉低。若按钮只有常闭 NC 触点，当前固件不能直接按此方式使用；需要先把输入逻辑改为常闭断线触发，再接线。

## LCD 连接

LCD 使用 SPI2 单向发送，当前没有触摸控制器：

```text
PA5  -> RST
PA6  -> BLK
PA7  -> CS
PB13 -> SCK
PB14 -> DC
PB15 -> SDA/MOSI
GND  -> GND
```

像素数据由 SPI2 TX DMA 逐行发送。启动显示封面，按板载 `PB8/BOOT0` KEY 切换到 RTC、QSPI、CAN、ADC 和故障状态页。

## RTC 与备用电池

- RTC 使用板载 32.768 kHz LSE，IWDG 使用独立 LSI。
- 主电源存在时，RTC 由开发板电源供电；没有纽扣电池也可以计时，但断电后不能保证保持。
- 外接 3 V 纽扣电池座连接开发板 `BAT/VBAT` 与 `GND`，极性必须按原理图确认。
- CR2032 电池座正极接开发板 `BAT/VBAT`，负极接 `GND`；不要接到 `3V3` 或 `5V`。装电池前先用万用表确认电池座输出约 3 V 且极性正确。
- 掉电保持验收需要设置时间、切断 USB-C 和其他主电源、等待后重新上电，再比较 RTC 是否连续。

## DMA 资源

| DMA | 模式 | 用途 |
|---|---|---|
| `DMA1_Channel1` | USART1 RX，循环，高优先级 | 调试命令连续接收 |
| `DMA1_Channel2` | USART1 TX，普通，中优先级 | 自检和遥测异步发送 |
| `DMA1_Channel3` | SPI2 TX，普通，低优先级 | LCD 像素逐行发送 |
| `DMA1_Channel4` | QUADSPI1，普通，中优先级 | QSPI 大块读取和逐页编程 |

ADC 当前每 100 ms 软件触发一次，不使用 DMA。QSPI 擦除由 Flash 命令执行，DMA 用于数据编程和读取。

## QSPI 存储布局

| 区域 | 起始地址 | 大小 | 用途 |
|---|---:|---:|---|
| 参数区 | `0x000000` | 64 KiB | 持久化配置和校准参数 |
| 日志区 | `0x010000` | 4032 KiB | 运行日志和诊断记录 |
| 升级包区 | `0x400000` | 4092 KiB | 后续 Bootloader/OTA 升级包暂存 |
| 人工测试区 | `0x7FF000` | 4 KiB | `qspi test confirm` 专用 |

测试区不得存放业务数据。当前仅完成物理分区和 DMA 访问能力，不代表参数双备份、日志格式、升级包签名或掉电事务已经实现。

## 首次电机上电顺序

1. 断开电机电源，逐根核对 PWM、编码器、ADC 和 GND。
2. 仅给 STM32 上电，确认启动自检正常且四路 PWM 为 0。
3. 测量编码器 A/B 高电平和 PA2 电压，确认不超过 STM32 输入范围。
4. 架空车轮并接通电机电源，先确认急停能够清零四路 PWM。
5. 使用低占空比依次验证左、右电机正反转和编码器计数方向。
6. 验证 200 ms 通信超时停车后，再开始低速 PID 整定。
