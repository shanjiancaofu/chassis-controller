# 裸机阶段进度

更新时间：2026-07-22

## 阶段目标

在引入 FreeRTOS 前，完成 STM32G474 底盘安全链路和板级外设的真实硬件验证，形成可重复构建、烧录、诊断和验收的裸机基线。

当前阶段不实现完整 OTA、Bootloader、位置环和复杂底盘运动学，不大规模调整现有目录。只补齐缺失驱动、自检入口、测试流程和必要文档。

## 当前结论

- STM32CubeIDE GCC 的 Debug、Release 和临时启用电机 Demo 的构建均为 `0 errors, 0 warnings`。
- D157B 与 STM32 接线已经完成，ADC 分压和采样链路通过万用表验证；左编码器方向系数调整为 `-1` 后重新烧录，手动向前转动时 `ld/rd` 均为正，向后转动时均为负。
- FDCAN2 当前固定为内部回环模式，内部回环通过不能代表 MCP2562FD 和外部 CAN 总线通过。
- USART DMA 收发、RTC 读取、QSPI JEDEC ID/8 MiB 容量检测和 FDCAN 内部回环已经通过实物运行验证。
- QSPI 保留扇区擦写、LCD 基础显示和 SPI DMA 已通过实物验证。FDCAN 外部真实总线仍是板级外设主要缺口。
- LCD 封面和动态状态页已通过实物显示验证，RTC、QSPI、CAN、ADC 和故障状态能够正确排版并持续刷新。
- 状态页中的 `QSPI PASS` 表示 JEDEC ID 和 8 MiB 容量校验通过，`CAN PASS` 表示 FDCAN 内部回环通过；ADC 显示当前采样换算值，已经完成万用表对照验证。
- KEY 消抖切页、IWDG 受控复位和 QSPI 1 KiB DMA 擦写校验均已通过实物验证。
- 当前 2 英寸屏只有 ST7789 显示接口，没有触摸控制器和触摸信号；现阶段使用板载 `PB8/BOOT0` 按键切换封面与状态页。

## 启动自检目标

启动自检不得转动电机，不得擦除 QSPI，也不得主动触发 IWDG 复位。目标输出如下：

```text
USART_TX_DMA: READY
USART_RX_DMA: READY
RTC_READ: PASS
RTC_BACKUP: READY
QSPI_ID: PASS
QSPI_RW_TEST: DISABLED
LCD: READY
FDCAN_INTERNAL: PASS
FDCAN_EXTERNAL: DISABLED
KEY: READY
ENCODER: READY
MOTOR: DISABLED
IWDG_RESET_TEST: DISABLED
```

没有硬件读回能力的项目不能只根据 HAL 返回值标记为 `PASS`。LCD、USART 物理链路、RTC 掉电保持、编码器和电机需要人工或外部仪器验收。

## 默认安全行为

- 电机 Demo 默认关闭，必须显式启用 `ENABLE_BAREMETAL_MOTOR_DEMO`。
- QSPI 启动时只允许读取 JEDEC ID，不得自动擦除或写入。
- QSPI 擦写测试必须使用固定保留扇区，并通过明确测试入口启用。
- QSPI 参数、日志和升级包分区不得使用最后的 `0x007FF000` 测试扇区。
- IWDG 复位测试默认关闭，只能通过 `iwdg reset confirm` 触发，并使用 RTC 备份寄存器和 RCC 复位标志共同确认复位原因。
- 无效、校验失败或不属于控制协议的 CAN 帧不得刷新命令超时。
- 急停属于软件清零 PWM，不等同于独立硬件断电急停。
