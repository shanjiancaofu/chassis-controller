# 裸机阶段进度

更新时间：2026-07-19

## 阶段目标

在引入 FreeRTOS 前，完成 STM32G474 底盘安全链路和板级外设的真实硬件验证，形成可重复构建、烧录、诊断和验收的裸机基线。

当前阶段不实现完整 OTA、Bootloader、位置环和复杂底盘运动学，不大规模调整现有目录。只补齐缺失驱动、自检入口、测试流程和必要文档。

## 状态说明

| 状态 | 含义 |
|---|---|
| 已实现 | 代码已经接入并通过编译，但不代表硬件验证通过 |
| 部分完成 | 已有基础配置或部分链路，仍缺关键接入 |
| 待实现 | 尚无可用代码路径 |
| READY | 初始化完成，等待外部设备、动作或人工验证 |
| PASS | 已通过固件自动校验或真实硬件验收 |
| DISABLED | 危险、破坏性或会引起运动的测试默认关闭 |

## 当前结论

- STM32CubeIDE GCC 的 Debug、Release 和临时启用电机 Demo 的构建均为 `0 errors, 0 warnings`。
- 电机控制、安全停车和 ADC 采样代码已经接入，但尚未完成真实驱动板、电机和线束验证。
- FDCAN2 当前固定为内部回环模式，内部回环通过不能代表 MCP2562FD 和外部 CAN 总线通过。
- QSPI、LCD 和 FDCAN 外部真实总线仍是裸机阶段主要缺口。
- 所有真实硬件验收项当前保持未勾选。

## 启动自检目标

启动自检不得转动电机，不得擦除 QSPI，也不得主动触发 IWDG 复位。目标输出如下：

```text
USART_TX_DMA: READY
USART_RX_DMA: READY
RTC_READ: PASS
RTC_BACKUP: READY
QSPI_ID: PASS
QSPI_RW_TEST: DISABLED
LCD: READY
FDCAN_INTERNAL: PASS
FDCAN_EXTERNAL: DISABLED
ENCODER: READY
MOTOR: DISABLED
IWDG_RESET_TEST: DISABLED
```

没有硬件读回能力的项目不能只根据 HAL 返回值标记为 `PASS`。LCD、USART 物理链路、RTC 掉电保持、编码器和电机需要人工或外部仪器验收。

## 默认安全行为

- 电机 Demo 默认关闭，必须显式启用 `ENABLE_BAREMETAL_MOTOR_DEMO`。
- QSPI 启动时只允许读取 JEDEC ID，不得自动擦除或写入。
- QSPI 擦写测试必须使用固定保留扇区，并通过明确测试入口启用。
- IWDG 复位测试默认关闭，触发后必须读取并记录复位原因。
- 无效、校验失败或不属于控制协议的 CAN 帧不得刷新命令超时。
- 急停属于软件清零 PWM，不等同于独立硬件断电急停。
