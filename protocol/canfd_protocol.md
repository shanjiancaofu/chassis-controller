# CAN FD 协议

当前协议用于 STM32G474 内部回环和首轮硬件联调。控制量和反馈量尚未接入，因此只冻结总线参数、报文方向和安全规则。

## 总线参数

- 帧格式：CAN FD，开启 BRS。
- 仲裁速率：500 kbps。
- 数据速率：2 Mbps。
- 标识符：11 位标准 ID。
- 字节序：多字节整数使用小端序。

## 节点

| 节点 | 职责 |
|---|---|
| Jetson | 下发运行模式和底盘控制命令 |
| STM32G474 | 执行控制、采集反馈并上报状态和故障 |

## 报文 ID

| ID 范围 | 方向 | 用途 | 当前状态 |
|---|---|---|---|
| `0x100`-`0x17F` | Jetson -> STM32 | 控制命令 | 等待执行器接口确定 |
| `0x180`-`0x1FF` | STM32 -> Jetson | 状态反馈 | 等待传感器接口确定 |
| `0x200`-`0x23F` | 双向 | 心跳和运行状态 | 待实现 |
| `0x240`-`0x27F` | STM32 -> Jetson | 故障和诊断 | 待实现 |
| `0x700`-`0x77F` | 双向 | 开发联调 | 仅开发阶段使用 |

正式报文采用固定 ID 和固定长度，不在同一个 ID 下复用多种数据布局。

## 通用规则

- 周期控制报文必须携带递增计数器，STM32 检测重复、乱序和长时间中断。
- 控制报文超时后必须进入安全状态，不能继续保持最后一次运动命令。
- 心跳只能说明节点运行，不能代替执行器状态反馈。
- 故障报文需要区分当前故障和历史故障。
- CAN FD 自带链路 CRC；是否增加端到端校验，在控制报文格式确定后决定。
- 协议不直接传输结构体内存，所有字段必须明确偏移、长度、单位和缩放比例。

## 内部回环报文

| 项目 | 值 |
|---|---|
| CAN ID | `0x721` |
| 方向 | STM32 自发自收 |
| DLC | 8 bytes |
| Payload | `43 48 41 53 53 49 53 01` |
| ASCII | `CHASSIS` + 测试版本 `0x01` |

该报文只验证 FDCAN 初始化、过滤器、中断、发送和接收链路，不属于正式车端协议。
